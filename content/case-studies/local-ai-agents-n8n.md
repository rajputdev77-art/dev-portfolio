## Context

I wanted to understand AI and automation from the inside, not just as a user of tools. Running models locally and building workflows in n8n gave me a way to learn by shipping — no cloud bills, full control, real systems running on my own machine.

## The Problem

Most "AI automation" tutorials are toy examples. The gap between a hello-world workflow and something that actually runs reliably every day is massive. I wanted to close that gap for myself.

## How I Approached It

Started with the smallest working version of each idea and layered complexity on top only after the simple version ran cleanly. Learned n8n node-by-node. Set up local model inference. Wired the two together. Treated every break as a design lesson, not a setback.
